Yul Glossary

Canonical terms for this workspace — use these words consistently.

← Lessons
Inline assembly
Yul embedded inside Solidity via assembly { … }. Solady ERC20 hot paths live here.
Scratch band
Fixed memory region 0x00–0x3f reused to build keccak256 inputs without allocating dynamic memory.
Namespaced seed
A pseudorandom constant (e.g. _BALANCE_SLOT_SEED) replacing compiler slot numbers to avoid inheritance storage collisions.
Big-endian mstore
Each mstore writes MSB-first within a 32-byte word. Addresses land at mem[offset+12..offset+31].
Hash offset 0x0c
Start keccak256 at byte 12 — skips ABI padding before an address, includes seed tail in a 32-byte preimage.
Clean (shr/shl)
shr(96, shl(96, addr)) — canonical address in low 160 bits. For comparisons and ecrecover checks.
Pack (shl/or)
or(shl(96, addr), seed) — address and seed in one word for a single mstore + hash.
Custom error selector
First 4 bytes of keccak256("ErrorName()"). Reverted via mstore(0x00, sel); revert(0x1c, 0x04).
Topic0
Event signature hash — first indexed topic in log3. Precomputed as _TRANSFER_EVENT_SIGNATURE etc.
log3
EVM opcode: emit event with 3 indexed topics + non-indexed data from memory.
Memory-safe-assembly
Solidity annotation telling the compiler the block respects free-memory-pointer rules at 0x40.
