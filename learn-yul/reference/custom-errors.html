Revert pattern

if gt(amount, fromBalance) {
    mstore(0x00, 0xf4d678b8)   // InsufficientBalance()
    revert(0x1c, 0x04)
}

Why revert(0x1c, 0x04)?

mstore is big-endian. A 4-byte selector in the low 32 bits of a word lands at the end of the 32-byte write:

mstore(0x00, 0xf4d678b8) mem: 0x00 ─────────────────────── 0x1b │ 0x1c 0x1d 0x1e 0x1f [ 28 zero bytes ] [ f4 d6 78 b8 ] └── revert(0x1c, 0x04) ──┘

Offset 0x1c + length 0x04 returns exactly the selector — no extra ABI encoding.
