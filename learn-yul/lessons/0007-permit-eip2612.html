EIP-2612 Permit in Yul

EIP-712 hashing, ecrecover, and nonce — the hardest block in Solady ERC20.

← All lessons · ← 0006 · Permit reference

What permit adds

permit(owner, spender, value, deadline, v, r, s) sets allowance via an off-chain signature — no approve transaction. Solady implements the full EIP-2612 + EIP-712 flow in one assembly block.

Primary source: EIP-2612 · Permit reference.

Step 1 — deadline and address cleaning

if gt(timestamp(), deadline) {
    mstore(0x00, 0x1a15a3cc)   // PermitExpired()
    revert(0x1c, 0x04)
}
owner := shr(96, shl(96, owner))
spender := shr(96, shl(96, spender))

Step 2 — load nonce

mstore(0x0c, _NONCES_SLOT_SEED)
mstore(0x00, owner)
let nonceSlot := keccak256(0x0c, 0x20)
let nonceValue := sload(nonceSlot)

Same Pattern A as balanceOf — different seed (_NONCES_SLOT_SEED = 0x38377508).

Step 3 — EIP-712 digest

At free memory pointer m := mload(0x40):

  1. Hash domain: keccak256(EIP712Domain typehash, nameHash, versionHash, chainid, address(this))
  2. Hash struct: keccak256(Permit typehash, owner, spender, value, nonce, deadline)
  3. Final digest: keccak256(0x1901 || domainSeparator || structHash)

Solady packs intermediate hashes into scratch at 0x2e and 0x4e before the final keccak256(0x2c, 0x42).

Step 4 — ecrecover

mstore(0x00, digest)
mstore(0x20, and(0xff, v))
mstore(0x40, r)
mstore(0x60, s)
let t := staticcall(gas(), 1, 0x00, 0x80, 0x20, 0x20)
if iszero(eq(mload(returndatasize()), owner)) {
    mstore(0x00, 0xddafbaef)   // InvalidPermit()
    revert(0x1c, 0x04)
}

Precompile address 1 = ecrecover. Clever nonce bump: sstore(nonceSlot, add(nonceValue, t)) — increments only when t = 1 (success).

Step 5 — set allowance + emit

// allowance slot — packed owner/spender layout
mstore(0x40, or(shl(160, _ALLOWANCE_SLOT_SEED), spender))
sstore(keccak256(0x2c, 0x34), value)
log3(add(m, 0x60), 0x20, _APPROVAL_EVENT_SIGNATURE, owner, spender)
