Events & log3

Precomputed topic hashes, indexed addresses, and amount in data.

Solidity event → EVM log

event Transfer(address indexed from, address indexed to, uint256 amount);

Compiles to:

  • topic0 — keccak256("Transfer(address,address,uint256)")
  • topic1, topic2 — indexed from, to (32-byte left-padded addresses)
  • data — non-indexed amount (32 bytes)

Solady precomputes topic0 as _TRANSFER_EVENT_SIGNATURE — never pays keccak at runtime for the signature string.

Primary source: Solidity events.

log3 in transfer

mstore(0x20, amount)
log3(0x20, 0x20, _TRANSFER_EVENT_SIGNATURE, caller(), shr(96, mload(0x0c)))
Argument Role
0x20, 0x20Data: 32-byte amount at mem[0x20..0x3f]
_TRANSFER_EVENT_SIGNATUREtopic0 — event hash
caller()topic1 — from
shr(96, mload(0x0c))topic2 — to recovered from scratch

Recovering addresses from scratch

After mstore(0x00, to) for the recipient balance slot, the address still sits at mem[0x0c..0x1f]. mload(0x0c) reads that word; shr(96, …) extracts the address for use as an indexed topic (left-padded by the LOG opcode).

Same trick in approve: shr(96, mload(0x2c)) reads the spender from allowance scratch.

Mint and burn topics

// _mint: from = address(0)
log3(0x20, 0x20, _TRANSFER_EVENT_SIGNATURE, 0, shr(96, mload(0x0c)))

// _burn: to = address(0)
log3(0x00, 0x20, _TRANSFER_EVENT_SIGNATURE, shr(96, shl(96, from)), 0)
